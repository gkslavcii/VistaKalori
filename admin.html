🟢 FatSecret API Ayarları

    
FatSecret Platform API entegrasyonu. Credentials Firestore'da saklanır, sunucu tarafındaki Cloud Function proxy ile çalışır (CORS sorunu yok).


    

      

        📋 Kurulum (3 Adım)

        1. platform.fatsecret.com/api → hesap oluştur → "Generate API Key" tıkla

        2. Aşağıya doğru key'leri gir, "Kaydet" butonuna bas

        3. Cloud Function'ı deploy et: firebase deploy --only functions
      

      

        ⚠️ DİKKAT — İki farklı key seti var, karıştırma!


        ✅ DOĞRU → "REST API OAuth 2.0" bölümündeki:

           • Client ID (uzun alfanumerik string)

           • Client Secret (sadece 1 kez gösterilir!)


        ❌ YANLIŞ → "REST API Consumer Key" ve "Consumer Secret" (bunlar OAuth 1.0 için, burada işe yaramaz)

🌐 FatSecret Sabit IP Kurulumu

    
FatSecret API, IP whitelist kullandığı için Cloud Functions'ın değişen IP'si sorun oluşturur. Çözüm: Sabit çıkış IP'si ayarlamak.


    

      📋 Adım 1: Cloud NAT ile Sabit IP (Önerilen)


      
      A) Google Cloud Console'dan:

      1. Console → VPC Network → IP Addresses → "Reserve External Static Address"

      2. Region: us-central1 (Cloud Functions region'ınız)

      3. IP adresini not alın (ör: 35.192.x.x)


      
      B) Cloud NAT yapılandır:

      4. Console → Network Services → Cloud NAT → "Create"

      5. Gateway name: fitsofra-nat

      6. Network: default

      7. Region: us-central1

      8. Cloud Router: Yeni oluştur → fitsofra-router

      9. NAT IP: "Manual" seç → Yukarıda oluşturduğun statik IP'yi ekle

      10. Source: "All" seç (Serverless VPC dahil)



      C) VPC Connector oluştur:

      11. Console → VPC Network → Serverless VPC Access → "Create Connector"

      12. Name: fitsofra-connector

      13. Region: us-central1

      14. Network: default

      15. IP range: 10.8.0.0/28 (boş bir subnet)


      
      D) Cloud Function'ı güncelle:

      16. firebase.json dosyasına ekle:
    


    
{
  "functions": {
    "source": "functions",
    "runtime": "nodejs18",
    "vpcConnector": "fitsofra-connector",
    "vpcConnectorEgressSettings": "ALL_TRAFFIC"
  }
}


    

      📋 Adım 2: FatSecret'a IP Tanımla


      17. platform.fatsecret.com → API ayarları → IP whitelist

      18. Sabit IP adresini ekle (ör: 35.192.x.x)

      19. Deploy et: firebase deploy --only functions


      
      ⚡ Alternatif (gcloud CLI ile):
    


    
## 1) Sabit IP ayır
gcloud compute addresses create fitsofra-ip \
  --region=us-central1 --project=fitsofra-51176

## IP adresini göster
gcloud compute addresses describe fitsofra-ip \
  --region=us-central1 --format="value(address)"

## 2) Cloud Router oluştur
gcloud compute routers create fitsofra-router \
  --network=default --region=us-central1

## 3) Cloud NAT oluştur (sabit IP ile)
gcloud compute routers nats create fitsofra-nat \
  --router=fitsofra-router --region=us-central1 \
  --nat-external-ip-pool=fitsofra-ip \
  --nat-all-subnet-ip-ranges

## 4) VPC Connector oluştur
gcloud compute networks vpc-access connectors create \
  fitsofra-connector --region=us-central1 \
  --network=default --range=10.8.0.0/28

## 5) Cloud Function'ı VPC ile deploy et
firebase deploy --only functions


    

      💡 Sonuç: Tüm Cloud Functions dış trafiği tek bir sabit IP'den çıkar.

      Bu IP'yi FatSecret whitelist'e ekleyince bir daha değişmez!

      Maliyet: ~$1-3/ay (sabit IP + Cloud NAT + VPC Connector)
